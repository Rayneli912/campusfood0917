# 新聞頁面增強功能測試指南

## 功能概述
用戶端新聞頁面（`/user/news`）已經進行了全面增強，實現了以下功能：
1. 整合後端資料庫和 localStorage 的消息內容
2. 添加照片縮圖顯示功能
3. 實現點擊單條消息的懸浮視窗功能
4. 支持即時數據同步

## 新功能特點

### 1. 數據源整合
- **後端數據**：從 Supabase `near_expiry_posts` 表載入已發布消息
- **本地數據**：從 localStorage 載入本地消息
- **智能合併**：自動去重並按日期排序（最新在前）
- **即時同步**：監聽資料庫變更和本地存儲變更

### 2. 增強的視覺設計
- **照片縮圖**：每條消息右上角顯示照片縮圖
- **懸停效果**：卡片懸停時顯示陰影效果
- **內容預覽**：消息內容限制在3行內（line-clamp-3）
- **響應式佈局**：適應不同螢幕尺寸

### 3. 懸浮視窗功能
- **大尺寸視窗**：最大寬度1000px，高度85vh
- **完整內容**：顯示完整消息內容和所有照片
- **照片網格**：多張照片以網格形式顯示
- **懸停效果**：照片支持縮放效果

### 4. 交互功能
- **點擊開啟**：點擊消息卡片開啟懸浮視窗
- **照片查看**：點擊縮圖開啟照片懸浮視窗
- **搜尋功能**：支持搜尋所有數據源的內容
- **即時更新**：數據變更時自動重新載入

## 測試步驟

### 1. 頁面導航測試
1. **從首頁進入**：
   - 在首頁「最新即食消息」區域點擊「查看更多」
   - 確認正確導航到 `/user/news` 頁面
   - 檢查頁面標題顯示「即食消息」

2. **直接訪問**：
   - 直接在瀏覽器輸入 `/user/news`
   - 確認頁面正常載入

### 2. 數據顯示測試

#### 測試數據載入
1. **檢查數據源**：
   - 確認頁面顯示來自後端的消息（ID以 `backend_` 開頭）
   - 確認頁面顯示來自本地的消息
   - 驗證消息按日期降序排列（最新在前）

2. **檢查內容顯示**：
   - 確認每條消息顯示標題、內容預覽、日期和來源
   - 檢查內容預覽是否限制在3行內
   - 驗證日期格式正確（年/月/日）

#### 測試照片顯示
1. **縮圖顯示**：
   - 檢查有照片的消息是否在右上角顯示縮圖
   - 確認多張照片時顯示 "+N" 指示器
   - 驗證沒有照片的消息不顯示縮圖

2. **縮圖點擊**：
   - 點擊照片縮圖開啟照片懸浮視窗
   - 確認不會觸發消息卡片的點擊事件
   - 測試多張照片的滑動功能

### 3. 懸浮視窗測試

#### 測試消息懸浮視窗
1. **開啟視窗**：
   - 點擊消息卡片的文字區域
   - 確認懸浮視窗正確開啟
   - 檢查視窗大小和位置

2. **內容顯示**：
   - 確認標題、日期、來源正確顯示
   - 檢查完整消息內容顯示
   - 驗證換行和格式正確

3. **照片顯示**：
   - 確認有照片的消息顯示「相關照片」區域
   - 檢查照片以網格形式顯示（桌面2列，手機1列）
   - 測試照片懸停縮放效果

#### 測試視窗交互
1. **關閉功能**：
   - 點擊 X 按鈕關閉視窗
   - 點擊視窗外區域關閉視窗
   - 按 ESC 鍵關閉視窗

2. **滾動功能**：
   - 測試長內容的滾動
   - 確認滾動條正常工作

### 4. 搜尋功能測試
1. **搜尋測試**：
   - 輸入關鍵字搜尋消息
   - 確認搜尋結果包含標題、內容、來源的匹配
   - 測試搜尋來自不同數據源的消息

2. **無結果處理**：
   - 輸入不存在的關鍵字
   - 確認顯示「沒有找到符合條件的即食消息」
   - 檢查提示文字正確顯示

### 5. 即時同步測試

#### 測試後端同步
1. **新增消息**：
   - 在管理後台新增並發布消息
   - 檢查新聞頁面是否即時更新
   - 確認新消息出現在列表頂部

2. **修改消息**：
   - 修改已發布消息的內容
   - 確認新聞頁面內容即時更新

#### 測試本地同步
1. **多標籤測試**：
   - 打開多個新聞頁面標籤
   - 在一個標籤修改 localStorage 數據
   - 檢查其他標籤是否同步更新

### 6. 響應式設計測試
1. **手機端測試**：
   - 切換到手機視圖
   - 確認卡片佈局適應小螢幕
   - 檢查懸浮視窗在手機上的顯示

2. **桌面端測試**：
   - 在大螢幕上測試
   - 確認懸浮視窗使用完整寬度
   - 檢查照片網格正確顯示

## 故障排除

### 常見問題
1. **數據不顯示**：
   - 檢查 Supabase 連接
   - 確認 localStorage 中有數據
   - 查看瀏覽器控制台錯誤

2. **照片不顯示**：
   - 檢查圖片 URL 是否有效
   - 確認 images 或 image_url 字段有值
   - 檢查網絡請求狀態

3. **懸浮視窗問題**：
   - 確認 Dialog 組件正確引入
   - 檢查 z-index 層級
   - 驗證狀態管理正確

4. **即時同步不工作**：
   - 檢查 Supabase 即時訂閱
   - 確認事件監聽器註冊
   - 查看控制台錯誤訊息

### 調試工具
- **Network 標籤**：檢查 API 請求
- **Application 標籤**：檢查 localStorage
- **Console 標籤**：查看錯誤訊息
- **Elements 標籤**：檢查 DOM 結構

## 技術實現細節

### 組件結構
```
NewsPage
├── 搜尋框
├── 新聞列表
│   ├── NewsCard (可點擊)
│   │   ├── 標題和縮圖
│   │   ├── 內容預覽
│   │   └── 日期和來源
│   └── NewsPhotoThumbnail
├── 消息懸浮視窗 (Dialog)
│   ├── 標題和描述
│   ├── 完整內容
│   └── 照片網格
└── 照片懸浮視窗 (PhotoViewerDialog)
```

### 數據流程
1. `loadCombinedNewsData()` 整合多數據源
2. 去重和排序處理
3. 設置即時監聽器
4. 響應用戶交互事件
5. 更新UI狀態

### 性能優化
- 使用 `useMemo` 優化搜尋過濾
- 正確清理事件監聽器
- 圖片懶載入
- 響應式圖片尺寸
