# 自動取消訂單功能測試指南

## 功能概述
當用戶於取餐倒計時結束時尚未完成取餐，系統會自動將訂單狀態變更為「已取消」，並在用戶端和店家端同步更新狀態。

## 測試步驟

### 1. 準備測試環境
1. 啟動應用程序
2. 登入用戶帳號和店家帳號（可使用不同瀏覽器標籤）
3. 確保有可用的商品和庫存

### 2. 創建測試訂單
1. 在用戶端下訂單
2. 在店家端接受訂單（狀態變為 `accepted`）
3. 在店家端標記「備餐完畢」（狀態變為 `prepared`）
4. 確認用戶端開始顯示10分鐘倒計時

### 3. 測試自動取消功能

#### 方法一：等待倒計時自然結束（需要10分鐘）
1. 等待倒計時結束
2. 觀察用戶端訂單狀態是否自動變為「已取消」
3. 檢查店家端訂單是否同步變為「已取消」
4. 確認訂單在店家儀表板的「已取消」欄位中顯示

#### 方法二：手動修改倒計時（快速測試）
1. 開啟瀏覽器開發者工具
2. 在 Console 中執行以下代碼來縮短倒計時：
```javascript
// 找到訂單ID（例如 order-001-20250908-001）
const orderId = "your-order-id-here";
const timerKey = `orderTimer_${orderId}`;

// 修改計時器，讓它在5秒後過期
const timerData = JSON.parse(localStorage.getItem(timerKey) || "{}");
const now = new Date().toISOString();
const fakeStartTime = new Date(Date.now() - 595 * 1000).toISOString(); // 595秒前開始

localStorage.setItem(timerKey, JSON.stringify({
  ...timerData,
  preparedAt: fakeStartTime,
  initialTime: 600,
  timeLeft: 5
}));

// 重新載入頁面讓變更生效
location.reload();
```

### 4. 驗證結果

#### 用戶端驗證
- [ ] 倒計時結束後，訂單狀態自動變為「已取消」
- [ ] 取消原因顯示為「逾時未取」
- [ ] 用戶收到相關提示訊息

#### 店家端驗證
- [ ] 訂單狀態同步變為「已取消」
- [ ] 訂單移動到「已取消」欄位
- [ ] 無法再點擊「完成訂單」按鈕
- [ ] 取消原因顯示為「逾時未取」

#### 庫存驗證
- [ ] 相關商品庫存已回補
- [ ] 缺貨商品重新上架（如果有庫存）

### 5. 邊界情況測試

#### 測試店家嘗試完成已取消訂單
1. 在訂單自動取消後
2. 嘗試在店家端點擊「完成訂單」按鈕
3. 確認系統顯示錯誤訊息：「此訂單已取消，無法完成。」

#### 測試逾時檢查
1. 當訂單已逾時但尚未自動取消時
2. 店家嘗試完成訂單
3. 確認系統自動取消訂單並顯示逾時訊息

### 6. 跨頁面同步測試
1. 同時開啟用戶端和店家端頁面
2. 讓訂單自動取消
3. 確認兩邊都即時更新狀態（無需手動重新載入）

## 預期行為總結

1. **倒計時結束**：用戶端倒計時歸零時自動觸發取消
2. **狀態同步**：用戶端和店家端訂單狀態同時變為「已取消」
3. **UI更新**：
   - 用戶端：顯示取消狀態和原因
   - 店家端：訂單移動到已取消欄位，完成按鈕消失
4. **庫存回補**：已扣除的商品庫存自動恢復
5. **操作限制**：店家無法再完成已取消的訂單
6. **事件廣播**：所有相關頁面即時收到狀態更新通知

## 故障排除

如果功能不正常，請檢查：
1. 瀏覽器控制台是否有錯誤訊息
2. localStorage 中的訂單數據是否正確
3. 事件監聽器是否正常註冊
4. 同步服務是否正確執行

## 技術實現細節

### 關鍵組件
- `use-order-timer.ts`：倒計時邏輯和自動取消觸發
- `active-order-tracking.tsx`：用戶端訂單追蹤顯示
- `sync-service.ts`：訂單狀態同步和庫存回補
- `store/orders/[id]/page.tsx`：店家端訂單詳情和操作限制

### 事件流程
1. 倒計時結束 → `orderExpired` 事件
2. 調用 `syncOrderStatus` 取消訂單
3. 廣播 `orderStatusUpdated` 事件
4. 所有監聽頁面更新UI
5. 庫存回補和狀態持久化
