# 管理員後台新聞 CRUD 功能修復測試指南

## 問題描述
用戶回報管理員後台中無法確實刪除和修改即食消息的問題。

## 修復方案

### 🔧 問題原因分析
1. **權限問題**：管理員後台直接使用 Supabase 客戶端，可能缺乏足夠的權限來修改 `near_expiry_posts` 表
2. **客戶端限制**：前端直接操作資料庫可能受到 RLS（Row Level Security）政策限制
3. **同步問題**：操作後沒有正確廣播更新事件

### 🛠️ 修復措施

#### 1. 創建專用 API 端點
- **`/api/news`** (POST)：創建新聞
- **`/api/news/[id]`** (PUT)：更新新聞  
- **`/api/news/[id]`** (DELETE)：刪除新聞

#### 2. 使用 Supabase Admin 客戶端
- API 端點使用 `supabaseAdmin` 而非客戶端 `supabase`
- 擁有完整的資料庫操作權限
- 繞過 RLS 限制

#### 3. 改進錯誤處理和調試
- 添加詳細的控制台日誌
- 改進錯誤訊息顯示
- 正確的異常處理流程

#### 4. 確保同步機制
- 操作成功後廣播 `NEWS_DATA_UPDATED` 事件
- 重新載入數據以反映變更
- 與用戶端即時同步

## 測試步驟

### 🧪 測試環境準備
1. **開啟開發者工具**：
   - 按 F12 或右鍵 → 檢查
   - 切換到 Console 標籤查看日誌
   - 切換到 Network 標籤監控 API 請求

2. **登入管理員後台**：
   - 確保以管理員身份登入
   - 導航到「新聞管理」頁面

### 📝 創建功能測試

#### 測試案例 1：創建新後端新聞
1. **操作步驟**：
   - 點擊「新增消息」按鈕
   - 填寫表單：
     - 標題：「測試新聞 - 創建功能」
     - 內容：「這是一條測試新聞，用於驗證創建功能」
     - 來源：「系統公告」
     - 狀態：「已發布」
     - 圖片 URL：（選填）
   - 點擊「發布」按鈕

2. **預期結果**：
   - 控制台顯示：`正在創建新聞: {title: "測試新聞...", ...}`
   - Network 標籤顯示 POST `/api/news` 請求
   - 返回狀態碼 201
   - 顯示成功 toast 訊息
   - 表格中出現新創建的新聞
   - 用戶端頁面即時顯示新新聞

3. **驗證方法**：
   - 檢查新聞是否出現在表格中
   - 確認 ID 以 `backend_` 開頭
   - 在用戶端首頁檢查是否顯示
   - 在 Supabase 控制台檢查資料庫記錄

### ✏️ 更新功能測試

#### 測試案例 2：更新後端新聞
1. **操作步驟**：
   - 找到一條 ID 以 `backend_` 開頭的新聞
   - 點擊編輯按鈕（鉛筆圖標）
   - 修改標題為：「已修改 - 測試更新功能」
   - 修改內容
   - 點擊「更新」按鈕

2. **預期結果**：
   - 控制台顯示：`正在更新後端新聞: {backendId: "123", ...}`
   - Network 標籤顯示 PUT `/api/news/123` 請求
   - 返回狀態碼 200
   - 顯示成功 toast 訊息
   - 表格中的新聞內容立即更新
   - 用戶端頁面即時反映變更

3. **驗證方法**：
   - 確認表格中顯示修改後的內容
   - 在用戶端檢查內容是否同步更新
   - 重新載入頁面確認變更持久化

#### 測試案例 3：更新本地新聞
1. **操作步驟**：
   - 找到一條不以 `backend_` 開頭的新聞
   - 點擊編輯按鈕
   - 修改內容
   - 點擊「更新」按鈕

2. **預期結果**：
   - 控制台顯示：`正在更新本地新聞: {id: "news123", ...}`
   - 控制台顯示：`本地新聞更新完成`
   - localStorage 中的數據更新
   - 表格和用戶端即時更新

### 🗑️ 刪除功能測試

#### 測試案例 4：刪除後端新聞
1. **操作步驟**：
   - 找到一條測試用的後端新聞
   - 點擊刪除按鈕（垃圾桶圖標）
   - 在確認對話框中點擊「刪除」

2. **預期結果**：
   - 控制台顯示：`正在刪除後端新聞: {backendId: "123", ...}`
   - Network 標籤顯示 DELETE `/api/news/123` 請求
   - 返回狀態碼 200
   - 顯示成功 toast 訊息
   - 新聞從表格中消失
   - 用戶端頁面即時移除該新聞

3. **驗證方法**：
   - 確認新聞不再出現在表格中
   - 在用戶端確認新聞已被移除
   - 在 Supabase 控制台確認記錄已刪除
   - 重新載入頁面確認刪除持久化

#### 測試案例 5：刪除本地新聞
1. **操作步驟**：
   - 找到一條本地新聞
   - 點擊刪除按鈕
   - 確認刪除

2. **預期結果**：
   - 控制台顯示：`正在刪除本地新聞: {id: "news123"}`
   - 控制台顯示刪除統計信息
   - localStorage 更新
   - 表格和用戶端即時更新

### 🔄 同步功能測試

#### 測試案例 6：跨頁面同步
1. **操作步驟**：
   - 同時打開管理員後台和用戶端首頁
   - 在管理員後台創建/修改/刪除新聞
   - 觀察用戶端頁面變化

2. **預期結果**：
   - 用戶端頁面即時反映所有變更
   - 無需手動刷新頁面
   - 變更在所有打開的標籤中同步

### 📊 錯誤處理測試

#### 測試案例 7：網絡錯誤處理
1. **操作步驟**：
   - 在開發者工具中模擬網絡離線
   - 嘗試創建/更新/刪除新聞
   - 觀察錯誤處理

2. **預期結果**：
   - 顯示友善的錯誤訊息
   - 不會導致頁面崩潰
   - 控制台記錄詳細錯誤信息

#### 測試案例 8：表單驗證
1. **操作步驟**：
   - 嘗試提交空白表單
   - 嘗試只填寫部分必填字段

2. **預期結果**：
   - 顯示驗證錯誤訊息
   - 阻止無效數據提交
   - 表單保持打開狀態

## 常見問題排除

### ❌ 問題：API 請求失敗
**可能原因**：
- Supabase 連接配置錯誤
- 環境變數未正確設置
- 網絡連接問題

**解決方法**：
1. 檢查 `.env.local` 文件
2. 確認 Supabase 項目狀態
3. 查看 API 端點錯誤日誌

### ❌ 問題：權限拒絕
**可能原因**：
- RLS 政策限制
- API 端點未使用管理員客戶端

**解決方法**：
1. 確認 API 端點使用 `supabaseAdmin`
2. 檢查 Supabase 表權限設置
3. 驗證 `SUPABASE_SERVICE_ROLE_KEY` 配置

### ❌ 問題：同步不工作
**可能原因**：
- 事件監聽器未正確註冊
- 廣播事件未觸發

**解決方法**：
1. 檢查 `NEWS_DATA_UPDATED` 事件廣播
2. 確認 Supabase 即時訂閱狀態
3. 重新載入頁面重置監聽器
