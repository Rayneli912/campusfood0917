# 管理員後台新聞管理功能測試指南

## 功能概述
管理員後台的「最新即食消息」功能已經完全重新設計，實現了以下核心功能：
1. 整合後端 Supabase 資料庫和 localStorage 的消息數據
2. 完整的 CRUD 操作（創建、讀取、更新、刪除）
3. 與用戶端即時同步
4. 支持照片 URL 管理
5. 發布狀態管理（已發布/草稿）

## 新功能特點

### 1. 數據源整合
- **後端數據**：從 Supabase `near_expiry_posts` 表載入所有消息
- **本地數據**：從 localStorage 載入本地消息
- **統一管理**：在同一界面管理所有數據源的消息
- **即時同步**：監聽資料庫變更和本地存儲變更

### 2. 完整的 CRUD 操作
- **創建消息**：新增消息直接保存到 Supabase
- **讀取消息**：顯示所有來源的消息（按日期排序）
- **更新消息**：根據消息來源更新到對應位置
- **刪除消息**：從對應數據源刪除消息

### 3. 增強的用戶界面
- **表格視圖**：清晰顯示標題、來源、日期、狀態、照片
- **載入狀態**：顯示數據載入進度
- **照片預覽**：表格中顯示縮圖，表單中顯示預覽
- **狀態標識**：清楚區分已發布和草稿狀態

### 4. 即時同步機制
- **資料庫監聽**：監聽 Supabase 即時變更
- **本地監聽**：監聽 localStorage 變更
- **跨頁面同步**：與用戶端頁面即時同步

## 測試步驟

### 1. 頁面載入測試
1. **訪問管理員後台**：
   - 登入管理員帳號
   - 導航到「新聞管理」頁面
   - 確認頁面正常載入

2. **數據顯示測試**：
   - 確認顯示來自後端的消息（ID以 `backend_` 開頭）
   - 確認顯示來自本地的消息
   - 驗證消息按日期降序排列
   - 檢查載入狀態正確顯示

### 2. 新增消息測試

#### 基本新增功能
1. **打開新增對話框**：
   - 點擊「新增消息」按鈕
   - 確認對話框標題顯示「新增即食消息」
   - 檢查表單字段為空白狀態

2. **填寫表單**：
   - 輸入標題（必填）
   - 輸入內容（必填）
   - 選擇來源（必填）
   - 選擇發布狀態（已發布/草稿）
   - 輸入圖片 URL（選填）

3. **表單驗證**：
   - 嘗試提交空表單，確認顯示錯誤訊息
   - 填寫必填字段後提交
   - 確認成功訊息顯示

#### 圖片功能測試
1. **圖片預覽**：
   - 輸入有效的圖片 URL
   - 確認表單中顯示圖片預覽
   - 測試無效 URL 的處理

2. **保存後驗證**：
   - 保存消息後檢查表格中是否顯示縮圖
   - 確認用戶端頁面也顯示相同圖片

### 3. 編輯消息測試

#### 編輯後端消息
1. **選擇後端消息**：
   - 點擊 ID 以 `backend_` 開頭的消息編輯按鈕
   - 確認對話框標題顯示「編輯即食消息」
   - 檢查表單預填現有數據

2. **修改並保存**：
   - 修改標題、內容、來源、狀態或圖片
   - 點擊「更新」按鈕
   - 確認成功更新訊息

3. **驗證同步**：
   - 檢查管理員後台列表更新
   - 確認用戶端頁面即時更新
   - 驗證 Supabase 資料庫中的數據

#### 編輯本地消息
1. **選擇本地消息**：
   - 點擊本地消息的編輯按鈕
   - 修改內容並保存
   - 確認 localStorage 中的數據更新

### 4. 刪除消息測試

#### 刪除確認流程
1. **打開刪除對話框**：
   - 點擊消息的刪除按鈕
   - 確認顯示 AlertDialog 確認對話框
   - 檢查消息標題正確顯示

2. **取消刪除**：
   - 點擊「取消」按鈕
   - 確認對話框關閉，消息保持不變

3. **確認刪除**：
   - 重新打開刪除對話框
   - 點擊「刪除」按鈕
   - 確認成功刪除訊息

#### 不同來源刪除測試
1. **後端消息刪除**：
   - 刪除 `backend_` 開頭的消息
   - 確認從 Supabase 中刪除
   - 驗證用戶端頁面同步移除

2. **本地消息刪除**：
   - 刪除本地消息
   - 確認從 localStorage 中移除
   - 檢查其他頁面同步更新

### 5. 搜尋和篩選測試
1. **搜尋功能**：
   - 輸入關鍵字搜尋標題
   - 測試內容搜尋
   - 測試來源搜尋
   - 確認搜尋結果正確

2. **來源篩選**：
   - 使用來源下拉選單篩選
   - 測試「全部來源」選項
   - 確認篩選結果準確

### 6. 即時同步測試

#### 跨頁面同步
1. **多標籤測試**：
   - 同時打開管理員後台和用戶端頁面
   - 在管理員後台新增消息
   - 確認用戶端即時顯示新消息

2. **編輯同步測試**：
   - 在管理員後台編輯消息
   - 檢查用戶端內容即時更新
   - 驗證狀態變更同步

3. **刪除同步測試**：
   - 在管理員後台刪除消息
   - 確認用戶端消息即時移除

#### 資料庫同步測試
1. **外部變更**：
   - 直接在 Supabase 控制台修改數據
   - 檢查管理員後台是否即時更新
   - 驗證用戶端同步更新

### 7. 狀態管理測試
1. **發布狀態**：
   - 創建草稿消息，確認不在用戶端顯示
   - 將草稿發布，確認用戶端顯示
   - 將已發布消息改為草稿，確認用戶端移除

2. **狀態標識**：
   - 確認表格中正確顯示狀態標章
   - 檢查已發布消息使用實心標章
   - 確認草稿消息使用空心標章

## 故障排除

### 常見問題
1. **數據不顯示**：
   - 檢查 Supabase 連接配置
   - 確認管理員權限設置
   - 查看瀏覽器控制台錯誤

2. **同步不工作**：
   - 檢查 Supabase 即時訂閱狀態
   - 確認事件監聽器正確註冊
   - 驗證網絡連接狀態

3. **操作失敗**：
   - 檢查必填字段是否完整
   - 確認圖片 URL 格式正確
   - 查看 API 請求錯誤訊息

4. **圖片不顯示**：
   - 確認圖片 URL 可訪問
   - 檢查 CORS 政策設置
   - 驗證圖片格式支持

### 調試工具
- **Network 標籤**：檢查 Supabase API 請求
- **Application 標籤**：檢查 localStorage 數據
- **Console 標籤**：查看錯誤和日誌訊息
- **Elements 標籤**：檢查 DOM 結構和樣式

## 技術實現細節

### 數據流程
1. `loadCombinedNewsData()` 整合多數據源
2. CRUD 操作根據 ID 前綴判斷數據源
3. Supabase 即時訂閱監聽變更
4. 事件廣播確保跨頁面同步

### 狀態管理
- 使用 React hooks 管理組件狀態
- 表單狀態與數據狀態分離
- 載入和錯誤狀態獨立管理

### 同步機制
- Supabase 即時訂閱：`postgres_changes`
- localStorage 事件監聽：`storage`
- 自定義事件廣播：`NEWS_DATA_UPDATED`

### 性能優化
- 使用 `useMemo` 優化篩選邏輯
- 正確清理事件監聽器
- 圖片懶載入和尺寸優化
