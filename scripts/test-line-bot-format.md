# LINE Bot 格式化訊息功能測試指南

## 📋 測試概述

本指南用於測試 LINE Bot 的格式化訊息處理功能，包括：
- 新增貼文格式解析
- 修改貼文格式解析  
- 貼文代碼生成和管理
- 前端界面顯示更新
- 管理員後台同步

## 🧪 測試環境

### 前置條件
- [x] Next.js 開發服務器運行中
- [x] Supabase 連接正常
- [x] LINE Bot Webhook 配置完成
- [x] 新欄位已添加到資料庫

## 🔧 第一階段：格式解析測試

### 1. 訪問格式測試頁面
```
http://localhost:3000/test/line-format
```

### 2. 測試有效的新增格式

#### 測試案例 1：完整格式
**輸入訊息**：
```
【地點】：圖書館一樓
【物品】：便當
【數量】：2
【領取期限】：今天下午 5 點前
【備註】：請聯絡 LINE ID: example
```

**預期結果**：
- ✅ 解析成功
- ✅ 地點：圖書館一樓
- ✅ 物品：便當
- ✅ 數量：2
- ✅ 期限：今天下午 5 點前
- ✅ 備註：請聯絡 LINE ID: example

#### 測試案例 2：最小必填格式
**輸入訊息**：
```
【地點】：學生餐廳
【物品】：麵包
【數量】：5
【領取期限】：
【備註】：
```

**預期結果**：
- ✅ 解析成功
- ✅ 地點：學生餐廳
- ✅ 物品：麵包
- ✅ 數量：5
- ✅ 期限：空（允許）
- ✅ 備註：空（允許）

### 3. 測試無效格式

#### 測試案例 3：缺少必填欄位
**輸入訊息**：
```
【地點】：
【物品】：便當
【數量】：2
【領取期限】：今天下午 5 點前
【備註】：請聯絡 LINE ID: example
```

**預期結果**：
- ❌ 解析失敗
- ❌ 錯誤：地點

#### 測試案例 4：數量格式錯誤
**輸入訊息**：
```
【地點】：圖書館一樓
【物品】：便當
【數量】：abc
【領取期限】：今天下午 5 點前
【備註】：請聯絡 LINE ID: example
```

**預期結果**：
- ❌ 解析失敗
- ❌ 錯誤：數量必須為 1-999 的數字

#### 測試案例 5：整體格式錯誤
**輸入訊息**：
```
地點：圖書館一樓
物品：便當
數量：2
```

**預期結果**：
- ❌ 解析失敗
- ❌ 錯誤：格式錯誤：必須包含完整的 5 行格式

### 4. 測試修改格式

#### 測試案例 6：有效修改格式
**輸入訊息**：
```
修改+ABC123
【地點】：圖書館二樓
【物品】：飲料
【數量】：3
【領取期限】：明天中午前
【備註】：已更新
```

**預期結果**：
- ✅ 解析成功
- ✅ 代碼：ABC123
- ✅ 內容解析正確

#### 測試案例 7：無效修改代碼格式
**輸入訊息**：
```
修改+INVALID123
【地點】：圖書館二樓
【物品】：飲料
【數量】：3
【領取期限】：明天中午前
【備註】：已更新
```

**預期結果**：
- ❌ 解析失敗
- ❌ 錯誤：修改指令格式錯誤
