# 增強版最新即食消息顯示功能測試指南

## 功能概述
用戶端首頁的「最新即食消息」區域已經進行了以下增強：
1. 擴大顯示面積，預設顯示六條訊息
2. 採用網格佈局（大螢幕時2列顯示）
3. 增大懸浮視窗尺寸和比例
4. 整合後端資料庫和 localStorage 的消息內容

## 新功能特點

### 1. 擴大顯示區域
- **增加內邊距**：從 `p-4` 增加到 `p-6`
- **增大標題**：從 `text-xl` 增加到 `text-2xl font-bold`
- **增加圖標尺寸**：從 `h-5 w-5` 增加到 `h-6 w-6`
- **更大的搜尋框**：高度從預設增加到 `h-10`

### 2. 網格佈局
- **響應式設計**：手機單列，大螢幕雙列顯示
- **顯示數量**：從3條增加到6條訊息
- **間距優化**：使用 `gap-3` 提供適當間距

### 3. 增強的懸浮視窗
- **更大尺寸**：從 `max-w-[600px]` 增加到 `max-w-[900px] lg:max-w-[1000px]`
- **更高比例**：從 `max-h-[80vh]` 增加到 `max-h-[85vh]`
- **更大字體**：標題使用 `text-xl font-bold`，內容使用 `text-base`
- **照片網格**：照片在中大螢幕上以2列顯示，高度增加到 `h-80`

### 4. 數據源整合
- **後端資料庫**：從 Supabase `near_expiry_posts` 表載入已發布的消息
- **本地存儲**：從 localStorage 載入本地消息
- **即時同步**：監聽資料庫變更和本地存儲變更
- **去重處理**：基於 ID 去除重複消息
- **智能排序**：按日期降序排列（最新在前）

## 測試步驟

### 1. 視覺檢查
1. **區域大小**：
   - 檢查「最新即食消息」區域是否比之前更大
   - 確認標題字體更大更醒目
   - 驗證搜尋框和選擇器高度一致

2. **佈局檢查**：
   - 在手機螢幕：確認消息以單列顯示
   - 在桌面螢幕：確認消息以雙列網格顯示
   - 驗證顯示6條消息（如果有足夠數據）

### 2. 懸浮視窗測試
1. **尺寸測試**：
   - 點擊任意消息進入詳情對話框
   - 確認對話框比之前更大
   - 檢查在不同螢幕尺寸下的顯示效果

2. **內容測試**：
   - 確認文字更大更易讀
   - 檢查照片以網格形式顯示（如果有多張）
   - 驗證照片懸停效果

### 3. 數據整合測試

#### 測試後端數據載入
1. **準備測試數據**：
   - 確保 Supabase 中有一些測試數據
   - 在 `near_expiry_posts` 表中添加幾條 `status = 'published'` 的記錄

2. **驗證載入**：
   - 重新載入首頁
   - 檢查是否顯示來自後端的消息
   - 確認消息 ID 以 `backend_` 開頭

#### 測試本地數據載入
1. **檢查 localStorage**：
   - 打開瀏覽器開發者工具
   - 查看 Application > Local Storage > news
   - 確認有本地消息數據

2. **驗證顯示**：
   - 確認本地消息也在列表中顯示
   - 檢查消息排序是否正確

#### 測試數據去重
1. **創建重複數據**：
   - 在後端和本地都創建相同 ID 的消息
   - 重新載入頁面

2. **驗證去重**：
   - 確認重複消息只顯示一次
   - 檢查顯示的是哪個版本的數據

### 4. 即時同步測試

#### 測試後端同步
1. **準備**：
   - 打開首頁並保持頁面開啟
   - 在另一個標籤頁打開管理後台

2. **測試新增**：
   - 在管理後台新增一條消息並發布
   - 檢查首頁是否即時更新顯示新消息

3. **測試修改**：
   - 修改已發布消息的內容
   - 確認首頁內容即時更新

#### 測試本地同步
1. **多標籤測試**：
   - 打開多個首頁標籤
   - 在一個標籤中修改 localStorage 的 news 數據
   - 檢查其他標籤是否同步更新

### 5. 搜尋和篩選測試
1. **搜尋功能**：
   - 輸入關鍵字搜尋消息
   - 確認搜尋結果正確
   - 測試搜尋來自不同數據源的消息

2. **來源篩選**：
   - 使用來源下拉選單篩選
   - 確認篩選結果包含所有數據源
   - 測試「全部來源」選項

3. **排序功能**：
   - 切換「最新到最舊」和「最舊到最新」
   - 確認排序正確應用到所有數據源

## 故障排除

### 常見問題
1. **後端數據不顯示**：
   - 檢查 Supabase 連接設置
   - 確認環境變數正確配置
   - 檢查資料庫中是否有 `status = 'published'` 的數據

2. **即時同步不工作**：
   - 檢查 Supabase 即時訂閱是否正常
   - 確認事件監聽器正確註冊
   - 查看瀏覽器控制台是否有錯誤

3. **佈局問題**：
   - 檢查 CSS 類名是否正確應用
   - 確認響應式斷點設置
   - 測試不同螢幕尺寸

### 調試工具
- 瀏覽器開發者工具 > Network：檢查 API 請求
- 瀏覽器開發者工具 > Application：檢查 localStorage
- 瀏覽器開發者工具 > Console：查看錯誤訊息

## 技術實現細節

### 數據載入流程
1. `loadCombinedNewsData()` 函數整合多個數據源
2. 優先載入後端數據，然後合併本地數據
3. 使用 ID 去重，按日期排序
4. 設置即時監聽器同步更新

### 響應式設計
- 使用 Tailwind CSS 的響應式前綴
- `grid-cols-1 lg:grid-cols-2` 實現響應式網格
- 懸浮視窗使用 `sm:max-w-[900px] lg:max-w-[1000px]`

### 性能優化
- 使用 React 的 `useMemo` 優化篩選邏輯
- 事件監聽器正確清理避免記憶體洩漏
- Supabase 訂閱在組件卸載時正確移除
