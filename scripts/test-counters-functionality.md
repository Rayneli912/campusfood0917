# 計數器功能測試指南

## 功能概述
網站瀏覽人次計數器系統包含兩個主要部分：
1. **首頁顯示**：自動顯示瀏覽人次和減少糧食浪費次數
2. **管理員後台**：可以手動管理和調整計數器數值

## 測試步驟

### 1. 首頁瀏覽人次測試
1. 開啟新的無痕瀏覽器視窗
2. 前往 http://localhost:3001
3. 檢查頁面底部是否顯示：
   ```
   網站瀏覽人次：X
   已減少 Y 次糧食浪費
   最後更新：[時間戳記]
   ```
4. 重新整理頁面，確認瀏覽人次**不會**重複增加（同一會話）
5. 開啟新的無痕視窗，瀏覽人次應該增加 1

### 2. 管理員後台計數器管理測試
1. 前往 http://localhost:3001/login
2. 使用 `guard`/`guard` 登入管理員後台
3. 在總覽頁面找到「計數器管理」區塊
4. 檢查是否顯示：
   - 計算模式切換（手動 / 自動）
   - 網站瀏覽人次調整
   - 糧食浪費次數調整
   - 立即增加按鈕

### 3. 手動模式測試
1. 確保計算模式為「手動」
2. 修改瀏覽人次數值，點擊「儲存數值」
3. 前往首頁檢查數值是否更新
4. 修改糧食浪費次數，點擊「儲存數值」
5. 前往首頁檢查數值是否更新

### 4. 自動模式測試
1. 切換到「自動（完成訂單數）」模式
2. 檢查是否顯示：
   - 目前完成訂單數
   - 偏移量設定
   - 目前顯示值計算
3. 調整偏移量，點擊「儲存偏移量」
4. 前往首頁檢查顯示值是否為：完成訂單數 + 偏移量

### 5. 跨頁面同步測試
1. 開啟兩個瀏覽器標籤頁
2. 一個顯示首頁，一個顯示管理員後台
3. 在管理員後台修改數值
4. 檢查首頁是否即時更新（可能需要幾秒鐘）

## 預期結果
- ✅ 首頁正確顯示計數器
- ✅ 每次新會話瀏覽人次增加 1
- ✅ 管理員後台可以調整數值
- ✅ 手動模式和自動模式都正常工作
- ✅ 跨頁面數據同步正常

## 故障排除
如果計數器不工作：
1. 檢查瀏覽器控制台是否有錯誤
2. 檢查 localStorage 中是否有 `siteCounters` 項目
3. 清除瀏覽器緩存和 localStorage 重新測試
