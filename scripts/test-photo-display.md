# 即食消息照片顯示功能測試指南

## 功能概述
在用戶端首頁的「最新即食消息」欄位中，每條訊息框的最後面會顯示一個方形照片框。用戶可以點擊照片查看懸浮視窗，支持多張照片滑動瀏覽。點擊單條訊息框進入完整訊息頁面時，會顯示完整照片而非縮圖。

## 功能特點

### 1. 照片縮圖顯示
- 在首頁最新即食消息中，每條消息右側顯示 48x48 像素的方形照片縮圖
- 如果有多張照片，縮圖右上角會顯示 "+N" 指示器（N 為額外照片數量）
- 只有包含照片的消息才會顯示照片框

### 2. 照片懸浮視窗
- 點擊縮圖會打開照片懸浮視窗
- 支持鍵盤導航（左右箭頭鍵、ESC 鍵）
- 顯示當前照片位置（如 "1 / 3"）
- 支持滑動瀏覽多張照片
- 點擊左右導航按鈕或使用指示器圓點切換照片

### 3. 完整消息頁面
- 點擊消息框（非照片區域）進入完整消息對話框
- 在對話框中顯示完整尺寸的照片
- 多張照片以垂直排列方式顯示

## 測試步驟

### 1. 準備測試環境
1. 啟動應用程序
2. 導航到用戶端首頁 (`/user/home`)
3. 確保「最新即食消息」區塊可見

### 2. 測試照片縮圖顯示
1. 檢查消息列表中是否顯示方形照片縮圖
2. 驗證多張照片的消息是否顯示 "+N" 指示器
3. 確認沒有照片的消息不顯示照片框

### 3. 測試照片懸浮視窗
1. **單張照片測試**：
   - 點擊只有一張照片的消息縮圖
   - 確認懸浮視窗打開並顯示照片
   - 確認沒有導航按鈕和指示器
   - 按 ESC 鍵或點擊關閉按鈕關閉視窗

2. **多張照片測試**：
   - 點擊有多張照片的消息縮圖
   - 確認懸浮視窗打開並顯示第一張照片
   - 測試左右導航按鈕功能
   - 測試鍵盤導航（左右箭頭鍵）
   - 測試底部指示器圓點點擊功能
   - 確認照片計數器正確顯示

3. **交互測試**：
   - 確認點擊照片縮圖不會觸發消息框的點擊事件
   - 測試懸浮視窗的響應式設計

### 4. 測試完整消息頁面
1. 點擊消息框的文字區域（避免點擊照片縮圖）
2. 確認完整消息對話框打開
3. 驗證照片以完整尺寸顯示
4. 確認多張照片垂直排列顯示
5. 測試對話框的滾動功能

### 5. 測試新聞頁面
1. 導航到 `/user/news` 頁面
2. 確認照片在新聞卡片中正確顯示
3. 驗證多張照片的垂直排列顯示

## 測試數據

系統已預設了包含照片的測試數據：

1. **news1**: 包含 2 張照片（應顯示 "+1" 指示器）
2. **news2**: 包含 1 張照片（單張照片，無指示器）
3. **news3**: 包含 1 張照片（單張照片，無指示器）

## 故障排除

### 常見問題
1. **照片不顯示**：檢查圖片 URL 是否正確，確認圖片文件存在
2. **懸浮視窗不打開**：檢查瀏覽器控制台是否有 JavaScript 錯誤
3. **導航不工作**：確認事件處理器正確綁定
4. **樣式問題**：檢查 CSS 類名是否正確應用

### 調試工具
- 使用瀏覽器開發者工具檢查元素結構
- 查看控制台錯誤訊息
- 檢查網絡請求確認圖片載入狀態

## 技術實現細節

### 組件架構
- `NewsPhotoThumbnail`: 照片縮圖組件
- `PhotoViewerDialog`: 照片懸浮視窗組件
- `getNewsImages()`: 輔助函數，處理照片數組邏輯

### 數據結構
```typescript
interface NewsItem {
  // ... 其他字段
  images?: string[]        // 多張照片數組
  image_url?: string | null // 兼容舊版單張照片
}
```

### 事件處理
- 縮圖點擊：`e.stopPropagation()` 防止觸發父元素事件
- 鍵盤導航：支持方向鍵和 ESC 鍵
- 觸摸支持：未來可擴展手勢滑動功能
